# گزیبو سمولیشن ماحول کو ترتیب دینا

**گزیبو (Gazebo)** ایک طاقتور اور وسیع پیمانے پر استعمال ہونے والا 3D روبوٹکس سمیلیٹر ہے۔ یہ آپ کو پیچیدہ روبوٹس اور ماحول کو ماڈل کرنے کی اجازت دیتا ہے، اور اس میں ایک اعلیٰ وفاداری کا طبیعیات کا انجن شامل ہے جو کشش ثقل، رگڑ، اور تصادم کی نقل کرتا ہے۔ ROS 2 ڈویلپرز کے لیے، گزیبو روبوٹ سافٹ ویئر کی جانچ اور ترقی کے لیے ایک ضروری ٹول ہے اس سے پہلے کہ اسے فزیکل ہارڈ ویئر پر تعینات کیا جائے۔

یہ گائیڈ ROS 2 کے ساتھ گزیبو کو انسٹال کرنے اور چلانے کی بنیادی باتیں بیان کرتی ہے۔

## انسٹالیشن

گزیبو ROS 2 کے ساتھ مضبوطی سے مربوط ہے۔ اسے انسٹال کرنے کا تجویز کردہ طریقہ `ros-dev-tools` اور گزیبو ROS پیکیج کے ذریعے ہے۔

زیادہ تر ROS 2 ڈسٹری بیوشنز (جیسے Humble یا Iron) کے لیے، آپ گزیبو کو درج ذیل کمانڈ کے ساتھ انسٹال کر سکتے ہیں:

```bash
sudo apt-get install ros-<distro>-gazebo-ros-pkgs
```
`<distro>` کو اپنی ROS 2 ڈسٹری بیوشن کے نام سے تبدیل کریں (مثلاً، `humble`)۔ یہ کمانڈ گزیبو خود، نیز ROS 2 کے ساتھ انٹرفیس کرنے کے لیے ضروری پلگ انز کو انسٹال کرتی ہے۔

## گزیبو آرکیٹیکچر

گزیبو دو اہم عمل کے طور پر چلتا ہے:

1.  **گزیبو سرور (`gzserver`)**: یہ ہیڈ لیس بیک اینڈ ہے جو طبیعیات کی سمولیشن چلاتا ہے، سینسر ڈیٹا تیار کرتا ہے، اور کمانڈز کو قبول کرتا ہے۔ آپ `gzserver` کو بغیر کسی گرافیکل انٹرفیس کے خود ہی چلا سکتے ہیں۔
2.  **گزیبو کلائنٹ (`gzclient`)**: یہ گرافیکل فرنٹ اینڈ ہے جو `gzserver` سے جڑتا ہے اور سمولیشن کو تصور کرتا ہے۔ آپ اسے ماحول کا معائنہ کرنے، ماڈلز کے ساتھ تعامل کرنے، اور سینسر آؤٹ پٹ دیکھنے کے لیے استعمال کر سکتے ہیں۔

## گزیبو کو لانچ کرنا

آپ درج ذیل کمانڈ کا استعمال کرکے گزیبو میں ایک سادہ، خالی دنیا لانچ کر سکتے ہیں:

```bash
gazebo
```

کسی مخصوص ورلڈ فائل کو لانچ کرنے کے لیے، آپ اسے ایک دلیل کے طور پر پاس کر سکتے ہیں:
```bash
gazebo worlds/empty.world
```

### ROS 2 کے ساتھ گزیبو کو لانچ کرنا

`gazebo_ros` پیکیج گزیبو شروع کرنے کے لیے ایک آسان لانچ فائل فراہم کرتا ہے۔ آپ اسے اس کے ساتھ چلا سکتے ہیں:

```bash
ros2 launch gazebo_ros gazebo.launch.py
```

یہ لانچ فائل گزیبو سرور اور کلائنٹ کو شروع کرتی ہے۔ آپ اسے دلائل بھی پاس کر سکتے ہیں۔ مثال کے طور پر، کسی مخصوص ورلڈ فائل کو لانچ کرنے کے لیے:

```bash
ros2 launch gazebo_ros gazebo.launch.py world:=/path/to/your/world.sdf
```

## گزیبو ورلڈ

ایک گزیبو **ورلڈ** کو **سمولیشن ڈسکرپشن فارمیٹ (SDF)** کا استعمال کرتے ہوئے ایک فائل میں بیان کیا جاتا ہے۔ ایک `.world` فائل (جو ایک SDF فائل ہے) سمولیشن میں ہر چیز کی وضاحت کرتی ہے، بشمول:
-   ماحول (روشنی، کشش ثقل)۔
-   جامد اشیاء (عمارتیں، فرنیچر، علاقہ)۔
-   روبوٹ ماڈلز۔
-   دنیا کی طبیعیات کی خصوصیات۔

یہ ایک سادہ ورلڈ فائل کا ایک اقتباس ہے:

```xml
<sdf version='1.7'>
  <world name='default'>
    <light name='sun' type='directional'>
      ...
    </light>
    <gravity>0 0 -9.8</gravity>
    <scene>
      <ambient>0.4 0.4 0.4 1</ambient>
      <background>0.7 0.7 0.7 1</background>
    </scene>
    <include>
      <uri>model://ground_plane</uri>
    </include>
    <include>
      <uri>model://sun</uri>
    </include>
  </world>
</sdf>
```

## ایک روبوٹ کو سپان کرنا

اپنے گزیبو سمولیشن میں ایک روبوٹ شامل کرنے کے لیے، آپ اسے "سپان" (spawn) کرتے ہیں۔ یہ عام طور پر ایک ROS 2 نوڈ کا استعمال کرتے ہوئے کیا جاتا ہے جو گزیبو کے ذریعے فراہم کردہ `/spawn_entity` سروس کو کال کرتا ہے۔

اس عمل میں عام طور پر شامل ہوتا ہے:
1.  آپ کے روبوٹ کا ماڈل URDF یا SDF فائل میں بیان کیا جانا۔
2.  ایک گزیبو ورلڈ شروع کرنا۔
3.  ایک لانچ فائل کا استعمال کرکے `spawn_entity.py` نوڈ کو چلانا، جو آپ کے روبوٹ کی تفصیل فائل کو پڑھتا ہے اور اسے سمولیشن میں شامل کرنے کے لیے گزیبو کو بھیجتا ہے۔

ہم اگلے سیکشن میں URDF اور SDF جیسے روبوٹ ڈسکرپشن فارمیٹس کا احاطہ کریں گے۔ گزیبو میں مہارت حاصل کرنا کسی بھی روبوٹکسٹ کے لیے بنیادی ہے، کیونکہ یہ آپ کے روبوٹ کے ڈیزائن اور کنٹرول الگورتھم پر کام کرنے کا ایک محفوظ، تیز، اور لاگت مؤثر طریقہ فراہم کرتا ہے۔
