# URDF اور SDF: آپ کے روبوٹ کی تفصیل

گزیبو میں روبوٹ کو نقل کرنے یا RViz میں اسے تصور کرنے کے لیے، آپ کو سب سے پہلے اس کی جسمانی ساخت کو بیان کرنے کا ایک طریقہ درکار ہے۔ ROS 2 اور گزیبو اس کے لیے دو بنیادی فارمیٹس استعمال کرتے ہیں: **یونیفائیڈ روبوٹ ڈسکرپشن فارمیٹ (URDF)** اور **سمولیشن ڈسکرپشن فارمیٹ (SDF)**۔

## URDF: ROS کے لیے معیار

**URDF** ایک XML پر مبنی فارمیٹ ہے جو ROS میں روبوٹ ماڈل کی حرکیات اور متحرکات کو بیان کرنے کے لیے استعمال ہوتا ہے۔ یہ سمولیشن سے باہر روبوٹس کو بیان کرنے کے لیے ایک معیار ہے۔

ایک URDF فائل روبوٹ کو **لنکس (links)** اور **جوائنٹس (joints)** کے درخت کے طور پر بیان کرتی ہے۔

-   **`<link>`**: ایک لنک روبوٹ کے ایک سخت جسمانی حصے کی نمائندگی کرتا ہے (مثلاً، بازو کا اگلا حصہ، ایک پہیہ، ایک دھڑ)۔ ہر لنک میں کمیت اور انرشیا جیسی جسمانی خصوصیات، نیز بصری اور تصادم کی خصوصیات ہوتی ہیں۔
    -   **`<visual>`**: لنک کی ظاہری شکل کی وضاحت کرتا ہے (یہ کیسا لگتا ہے)۔ یہ اکثر ایک 3D میش فائل (جیسے `.dae` یا `.stl`) ہوتی ہے۔
    -   **`<collision>`**: لنک کی تصادم جیومیٹری کی وضاحت کرتا ہے (تصادم کی جانچ کے لیے اس کی جسمانی شکل)۔ یہ ایک سادہ شکل (جیسے ایک باکس یا سلنڈر) یا ایک میش ہو سکتی ہے۔
    -   **`<inertial>`**: لنک کی متحرک خصوصیات کی وضاحت کرتا ہے، بشمول اس کی کمیت اور انرشیا میٹرکس۔

-   **`<joint>`**: ایک جوائنٹ دو لنکس کو آپس میں جوڑتا ہے اور ان کی نسبتی حرکت کی وضاحت کرتا ہے۔
    -   **`parent` اور `child`**: بتاتا ہے کہ کون سے دو لنکس کو جوڑا جا رہا ہے۔
    -   **`type`**: جوائنٹ کے ذریعے اجازت دی گئی حرکت کی قسم۔ عام اقسام میں شامل ہیں:
        -   `revolute`: ایک ہنج جوائنٹ جو ایک ہی محور کے گرد گھومتا ہے (جیسے کہ کہنی)۔
        -   `continuous`: ایک ریوولٹ جوائنٹ جس کی زاویہ کی کوئی حد نہیں ہوتی (جیسے ایک پہیہ)۔
        -   `prismatic`: ایک سلائیڈنگ جوائنٹ جو ایک ہی محور کے ساتھ حرکت کرتا ہے۔
        -   `fixed`: دو لنکس کے درمیان ایک سخت کنکشن۔

**مثال URDF سنیپٹ:**
```xml
<robot name="simple_arm">
  <!-- Base Link -->
  <link name="base_link">
    ...
  </link>

  <!-- Arm Link -->
  <link name="arm_link">
    ...
  </link>

  <!-- Joint connecting base to arm -->
  <joint name="base_to_arm_joint" type="revolute">
    <parent link="base_link"/>
    <child link="arm_link"/>
    <origin xyz="0 0 0.5"/>
    <axis xyz="0 1 0"/>
    <limit effort="1000.0" lower="-1.57" upper="1.57" velocity="0.5"/>
  </joint>
</robot>
```

### URDF کی حدود

URDF ایک واحد روبوٹ کی حرکیات کو بیان کرنے کے لیے بہترین ہے، لیکن اس کی حدود ہیں:
-   یہ ایک بند لوپ چین (ایک کائنمیٹک لوپ) کی نمائندگی نہیں کر سکتا۔
-   اسے دنیا میں غیر روبوٹک اشیاء (جیسے لائٹس، میزیں، یا عمارتیں) کو بیان کرنے کے لیے استعمال نہیں کیا جا سکتا۔
-   یہ سینسر ماڈلز کو براہ راست مخصوص کرنے کے لیے محدود سپورٹ رکھتا ہے۔

## SDF: گزیبو کی زبان

**SDF** گزیبو کے لیے مقامی فارمیٹ ہے۔ یہ بھی ایک XML پر مبنی فارمیٹ ہے، لیکن یہ URDF کا ایک سپر سیٹ ہے۔ SDF کو ایک سمولیشن کے بارے میں ہر چیز کو بیان کرنے کے لیے ڈیزائن کیا گیا ہے، نہ کہ صرف ایک واحد روبوٹ۔

SDF کی اہم خصوصیات:
-   **ورلڈ ڈسکرپشن (World Description)**: ایک پوری دنیا کو بیان کر سکتا ہے، بشمول روبوٹس، جامد اشیاء، لائٹس، طبیعیات کی خصوصیات، اور بہت کچھ۔
-   **بند لوپس کے لیے سپورٹ (Support for Closed Loops)**: پیچیدہ میکانزم کو ماڈل کر سکتا ہے جو URDF نہیں کر سکتا۔
-   **تفصیلی سینسر ماڈلز (Detailed Sensor Models)**: سینسرز (کیمرے، LiDAR، IMUs) اور ان کے شور ماڈلز کی وضاحت کے لیے ٹیگز کا ایک بھرپور سیٹ فراہم کرتا ہے۔
-   **پلگ ان کی وضاحت (Plugin Specification)**: آپ کو یہ مخصوص کرنے کی اجازت دیتا ہے کہ کسی دیے گئے ماڈل یا سینسر کے لیے کون سے پلگ ان لوڈ کیے جائیں۔

آپ SDF کو سمولیشن کے لیے ایک زیادہ مکمل زبان سمجھ سکتے ہیں۔ جبکہ URDF بیان کرتا ہے کہ *آپ کا روبوٹ کیا ہے*، SDF بیان کرتا ہے کہ *آپ کا سمولیشن کیا ہے*۔

## URDF بمقابلہ SDF: کون سا استعمال کریں؟

-   **URDF کو اپنے بنیادی روبوٹ ڈسکرپشن فارمیٹ کے طور پر استعمال کریں**۔ یہ ROS ایکو سسٹم میں ایک معیار ہے، اور RViz اور MoveIt جیسے ٹولز اس پر انحصار کرتے ہیں۔
-   **اپنے گزیبو ورلڈ فائلوں (`.world`) کے لیے SDF استعمال کریں**۔
-   جب آپ کو اپنے روبوٹ ماڈل میں گزیبو کی مخصوص خصوصیات (جیسے سینسر یا پلگ ان) شامل کرنے کی ضرورت ہو، تو آپ اپنی URDF فائل کے اندر خصوصی `<gazebo>` ٹیگز شامل کر سکتے ہیں۔ ROS ٹولز ان ٹیگز کو نظر انداز کر دیں گے، لیکن گزیبو انہیں پارس کرے گا۔

**مثال: URDF لنک میں گزیبو کلر ٹیگ شامل کرنا**
```xml
<link name="my_link">
  ...
  <gazebo reference="my_link">
    <material>Gazebo/Red</material>
  </gazebo>
</link>
```

پیچیدہ سمولیشنز کے لیے، آپ اپنے URDF کو ایک SDF فائل میں براہ راست تبدیل کر سکتے ہیں جب آپ روبوٹ کو گزیبو میں سپان کرتے ہیں۔ `gazebo_ros` پیکیج خود بخود ایسا کرنے کے لیے ٹولز فراہم کرتا ہے۔ یہ نقطہ نظر آپ کو اپنے روبوٹ کے لیے ایک واحد، کینونیکل URDF کو برقرار رکھنے کی اجازت دیتا ہے جبکہ سمولیشن کے لیے SDF کی مکمل طاقت کا فائدہ بھی اٹھاتا ہے۔
